# Library Database Analysis Using SQL

## Project Overview

The Library Database Analysis project is designed to manage and analyze library operations using MySQL. The system stores and manages information related to books, authors, publishers, borrowers, library branches, book copies, and book loans. It helps automate library processes, improve record management, and generate valuable insights through SQL queries.

## Objectives

* Design a relational database for library management.
* Manage books, authors, publishers, and borrowers efficiently.
* Track book availability across different library branches.
* Monitor book issue and return activities.
* Generate reports to support data-driven decision-making.

## Database Design

The project consists of 7 interconnected tables:

1. **tbl_book** – Stores book details.
2. **tbl_book_authors** – Stores author information for books.
3. **tbl_publisher** – Contains publisher details.
4. **tbl_library_branch** – Stores library branch information.
5. **tbl_borrower** – Maintains borrower records.
6. **tbl_book_copies** – Tracks book copies available at each branch.
7. **tbl_book_loans** – Records book borrowing and return transactions.

* Managing large volumes of book records.
* Tracking book availability across branches.

## Key Insights

* Centralized database management improves operational efficiency.
* Relational database design reduces data redundancy and maintains consistency.
* Real-time tracking of inventory improves resource management.
* SQL queries enable quick reporting and analysis.
* Branch-wise inventory monitoring supports better book distribution.
